# khub 3.0 路线图

## 3.0 — 产品成熟度提升（A）

**目标**: 从功能完整走向产品成熟
- 生产环境 bug 修复与边界情况回溯
- 性能压测 + 5000 并发基准
- 安全审计 + 渗透测试 + compliance 增强
- 用户体验打磨（Web UI / 桌面 / 小程序 / Flutter 统一设计语言）
- 文档全量校对 + 用户手册中文/英文版

## 3.1 — FastAPI 全面迁移（B）

**目标**: 全部路由迁移至 FastAPI，下线旧 API
- 逐个领域迁移：文档 → 临床 → 运营 → 课程 → KG → AI → 远程医疗 → 社区 → 公众号
- WebSocket 原生支持（替代 SSE 兼容层）
- ASGI Middleware 重写鉴权/CORS/限流
- 旧 khub/api.py 归档

## 3.2 — AI Agent 深化（C）

**目标**: 从 Agent 框架到 Agent 平台
- 可视化 Agent 构建器（拖拽式工具链）
- Agent 市场（模板上传/评分/使用统计）
- 自适应 Agent（历史表现自动优化）
- Agent 监控（执行日志/成功率/耗时统计）

## 3.3 — 商业化运营（D）

**目标**: 代码→产品→收入闭环
- 租户订阅计划（free/pro/enterprise）
- 用量计量 + 配额限制
- 运营管理后台
- 微信支付/支付宝对接

## 3.4 — 新行业复制（E）

**目标**: 垂直行业模板化快速复制
- veterinary/ — 兽医中医（动物患者/物种特化）
- beauty/ — 美容中医（皮肤辨证/方剂外用）
- sports/ — 体育康复（运动损伤/康复计划）
