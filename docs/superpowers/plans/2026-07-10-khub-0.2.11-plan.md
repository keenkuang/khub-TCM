## 实现说明

### A1：修复端口双启动 bug

`cli.py desktop` 子命令：当 `--electron` 时，不再启动 Python 后端（由 Electron `main.js` 内嵌的 `startKhub()` 管理）。

### A2：main.js 增强

- 添加实际托盘图标（可用 Base64 内联 PNG 或文件）
- 添加系统菜单（文件、帮助）
- 添加 Ollama 检测：`startKhub()` 完成后，尝试 `fetch('http://127.0.0.1:11434/api/tags')`，成功则设环境变量启动后端
- 添加 `electron-builder` 配置文件

### B1：3 张业务表

遵循 clinical v2 / course 模式追加。

### B2：微信 API 模块

Token 管理：appid+secret → `/cgi-bin/token` 获取 access_token，缓存 + 过期前 120 秒自动刷新。
API 调用封装：`_get(url)` / `_post(url, data)`，自动注入 access_token。
素材上传：`POST /cgi-bin/material/add_news`（图文）、`POST /cgi-bin/material/add_material`（图片）。
群发：`POST /cgi-bin/message/mass/sendall`（按标签/全部）、`POST /cgi-bin/message/mass/send`（按 openid）。
粉丝同步：`GET /cgi-bin/user/get`（分页拉取 openid 列表）→ `POST /cgi-bin/user/info.batchget`（批量拉取详情）。

### B3：CRUD 模块

按 ops 轻量模式：单文件 `store.py` 含 init/CRUD/回放。
`add_article` / `list_articles` / `schedule_publish` / `scan_due_schedules` / `sync_followers`。
容量检查: article submit 前检查必填字段。
