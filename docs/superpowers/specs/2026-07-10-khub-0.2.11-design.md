# 0.2.11 设计说明 — 桌面体验增强 + 微信公众号发布系统

## 线 A：桌面体验增强

### 现有状态
- `desktop/main.js` 90 行：Electron 壳（窗口/托盘/内嵌后端），缺托盘图标、无 IPC、无打包
- `cli.py --electron` 与 `main.js` 的 `startKhub()` **端口双启动 bug**
- 无离线模式、无本地模型集成

### MVP 范围

| 条目 | 说明 |
|------|------|
| 修复端口双启动 bug | `cli.py --electron` 启动 Electron 后不再自己启动后端，由 Electron 内嵌的后端统一管理 |
| 托盘图标 | 添加实际图标资源（SVG/PNG），从空白图标变为可见 |
| 系统菜单 | 文件菜单（打开本地库、退出）、帮助菜单（版本、关于） |
| 启动脚本 | 检测 Python 环境 + 自动 `pip install`，降低首次使用门槛 |
| electron-builder 打包 | 新增打包配置，支持 `.AppImage`/`.deb`（Linux） |
| Ollama 本地模型 | Electron 内嵌 Ollama 检测与启动（`fetch('http://127.0.0.1:11434')` 检测 + 环境变量 `KHUB_LLM_URL=http://127.0.0.1:11434` 自动配置） |

### 不在范围
- macOS/.dmg 分发、Windows/.exe 分发
- auto-updater 自动更新
- Service Worker 离线缓存
- 全局快捷键

## 线 B：微信公众号发布系统

### 表结构

**wechat_articles**（图文素材）：id, doc_id, title, author, digest, content, content_source_url, thumb_media_id, need_open_comment, wechat_media_id, wechat_url, status(draft/submitted/published/failed)

**wechat_schedules**（发布排期）：id, article_id, publish_at, tag_id, status(pending/publishing/published/failed)

**wechat_followers**（粉丝）：openid PK, nickname, subscribe, city, province, subscribe_time, tagid_list

### 模块

`khub/adapters/_wechat_auth.py` — Token 管理器（仿 `_feishu_auth.py`）
`khub/wechat/store.py` — 文章/排期/粉丝 CRUD
`khub/wechat/api.py` — 微信公众号平台 API 调用（素材上传/群发/粉丝同步）
`khub/wechat/__init__.py` — 空包标记

### 接口

- CLI：`khub wechat article-add`、`wechat article-list`、`wechat schedule`、`wechat publish`、`wechat sync-followers`
- REST：`POST /api/wechat/articles`、`GET /api/wechat/articles`、`POST /api/wechat/publish`、`GET /api/wechat/followers`
- Web UI：公众号面板（文章列表/新建/排期/粉丝概况）

### 与调度器集成

`scheduler.py` 的 YAML 任务可调用 `khub wechat publish --due` 扫描到期排期并自动发布。
