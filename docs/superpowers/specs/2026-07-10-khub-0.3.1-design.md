# khub 0.3.1 设计说明 — RBAC 权限框架

## 权限模型

基于 roles.md 定义的 7 角色，采用 **resource:action** 模型：

```python
PERMISSIONS = {
    "admin":    {"patients": "crud", "records": "crud", "consultations": "crud",
                 "appointments": "crud", "courses": "crud", "exam": "crud",
                 "docs": "crud", "users": "crud", "system": "admin"},
    "doctor":   {"patients": "crud", "records": "crud", "consultations": "crud",
                 "appointments": "crud", "courses": "r", "exam": "crud",
                 "docs": "r"},
    "nurse":    {"patients": "r", "records": "rw", "consultations": "r",
                 "appointments": "crud", "docs": "r"},
    "intern":   {"patients": "r", "records": "rw--draft", "consultations": "r--draft",
                 "appointments": "r", "docs": "r"},
    "receptionist": {"patients": "r", "appointments": "crud", "docs": "r"},
    "patient": {"patients": "r--self", "records": "r--self", "consultations": "r--self",
                "appointments": "crud--self", "docs": "r"},
    "guardian": {"patients": "r--guardian", "appointments": "crud--guardian"},
    "security": {"patients": "r--limited"},
}
```

### 权限检查函数
```python
check_permission(user, resource, action) -> bool
# user = {"role": "doctor", "user_id": 1}
# resource = "patients", action = "read"
```

## 文件结构

| 文件 | 操作 | 职责 |
|------|------|------|
| `khub/auth.py` | 修改 | 追加 `PERMISSIONS` 字典、`check_permission()`、`list_users()`、`update_user_role()` |
| `khub/api.py` | 修改 | API 端点级权限检查（`check_permission(current_user, "appointments", "read")`） |
| `khub/web/index.html` | 修改 | 导航栏加「用户管理」面板（仅 admin 可见） |
| `khub/web/script.js` | 修改 | 用户管理 UI：用户列表/创建/修改角色 |
| `khub/cli.py` | 修改 | `khub user-list` / `user-create` / `user-role` |
| `docs/config.md` | 修改 | 补充权限说明 |
| `tests/test_auth.py` | 修改 | 新增权限检查测试 |

## 实施要点

1. `check_permission` 在 `dispatch()` 入口处对每个请求调用（资源从路径推断，action 从 method 推断）
2. `admin` 角色跳过所有检查
3. 登录/登出/用户管理端点始终可访问
4. 暂不实现数据级隔离（留 0.3.2）
