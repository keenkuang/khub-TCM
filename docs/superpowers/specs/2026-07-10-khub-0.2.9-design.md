## 1. 目标与范围

在 v0.2.8（运营界面+可观测性）基础上，两条独立线并行推进。

### 线 A：RAG 检索增强生成
完善现有 `RAGEngine` 并接入问诊助手，使知识检索→上下文拼接→LLM 生成端到端可用。

### 线 B：知识库增强
新增标签/收藏系统、Markdown 后端渲染、批量导入进度。

### 不在范围内
- 多模态 RAG（图片/PDF 图表）
- 全文重新索引
- 课程管理系统（留 0.2.10）

## 2. 线 A：RAG 增强

### 2.1 上下文截断完善

**现状**：`RAGEngine._assemble_context` 只用 `per_doc = max(400, max_chars // len(sources))` 单层等分截断。

**完善**：增加第二层「从低分文档逐步缩短」和第三层「至少保留 100 字符」底线：

```python
def _assemble_context(self, sources: list[dict], max_chars: int = 3000) -> str:
    if not sources:
        return ""
    # 按分数降序
    sources.sort(key=lambda s: s.get("score", 0), reverse=True)
    per_doc = max_chars // len(sources)
    chunks = []
    for i, s in enumerate(sources):
        text = (s.get("content") or s.get("text") or "")[:per_doc]
        if not text.strip():
            continue
        chunks.append(f"[来源{i+1}] {text.strip()}")
    result = "\n\n".join(chunks)
    # 第二层：如果总体超出，从低分来源回缩
    while len(result) > max_chars and len(sources) > 1:
        sources.pop()
        per_doc = max_chars // max(len(sources), 1)
        chunks = []
        for i, s in enumerate(sources):
            text = (s.get("content") or s.get("text") or "")[:max(per_doc, 100)]
            if not text.strip():
                continue
            chunks.append(f"[来源{i+1}] {text.strip()}")
        result = "\n\n".join(chunks)
    return result
```

### 2.2 来源过滤 / 检索范围控制

`RAGEngine.ask(query, source_filter=None, k=5)`：
- `source_filter`：可选字符串/列表，仅检索指定来源（如 `"obsidian"`、`["quip"， "feishu"]`）
- `k`：返回 Top-K 来源数

后端 `Retriever.search_similar` 现有 `limit` 参数，加上 `source_filter` 参数过滤 `documents.source_ids`。

### 2.3 RAGEngine 接入问诊助手

`khub/clinical/consult_chat.py` 的 `chat()` 函数当前手拼 prompt（孪生摘要 + 历史 + 用户消息），改为：

```python
from ...llm.rag import RAGEngine
engine = RAGEngine(store)
rag_sources = engine._fetch_sources(user_msg, k=3)
rag_context = engine._assemble_context(rag_sources, max_chars=1500)
prompt = (
    f"你是一名中医问诊助手。\n\n"
    f"患者健康摘要：\n{summary}\n\n"
    f"相关文档知识：\n{rag_context}\n\n"
    f"对话历史：\n{history}\n\n"
    f"患者提问：{user_msg}"
)
```

### 2.4 增强 `/ask` 端点

- 支持 `source_filter` 参数（当前已处理 `body.get("source_filter")` 但未传入 RAGEngine）
- 返回 sources 增强为含文档标题、链接：
```json
{
  "reply": "...",
  "sources": [
    {"title": "伤寒论-太阳篇", "doc_id": "abc", "score": 0.92, "source": "library"},
    {"title": "桂枝汤方剂", "doc_id": "def", "score": 0.85, "source": "quip"}
  ]
}
```

## 3. 线 B：知识库增强

### 3.1 标签系统

**新增表**（接 WAL）：
```sql
doc_tags(id INTEGER PRIMARY KEY, doc_id TEXT, tag TEXT, created_at TEXT,
  UNIQUE(doc_id, tag))
```

**函数**（`khub/db.py` 或新模块 `khub/tags.py`）：
- `add_tag(store， doc_id, tag)` / `remove_tag(store， doc_id, tag)` / `list_tags(store)` / `get_doc_tags(store, doc_id)`

**REST**：
- `POST /documents/{id}/tags` `{"tag": "中医"}`
- `DELETE /documents/{id}/tags?tag=中医`
- `GET /tags` — 列出所有标签及其文档计数
- `GET /search?tag=中医` — 按标签筛选（扩展现有搜索）

**Web UI**：
- 文档详情页显示已有标签（tag 圆角徽章）
- 输入框添加新标签（回车提交）
- 标签旁 × 按钮删除
- 搜索栏来源筛选下拉旁增加标签下拉筛选

### 3.2 收藏/书签

**新增表**：
```sql
favorites(id INTEGER PRIMARY KEY, doc_id TEXT, created_at TEXT, UNIQUE(doc_id))
```

**函数**：`toggle_favorite(store， doc_id)` / `list_favorites(store)` / `is_favorite(store, doc_id)`

**REST**：`POST /documents/{id}/favorite`（toggle）、`GET /favorites`

**Web UI**：搜索结果/文档详情页的收藏星标按钮（☆/★）

### 3.3 Markdown 后端渲染

**改动**：
- `khub/api.py` 的 `GET /documents/{id}` 中，若 `format == 'markdown'`，用 `markdown` 库将 `content` 渲染为 HTML（安全清理后写入 `content`）
- `markdown` 为可选依赖（`pyproject.toml` 新增 `[project.optional-dependencies] md = ["markdown>=3.4"]`）
- 无 `markdown` 库时退回原样返回（前端 `esc()` 显示）
- Web UI 中 format=='markdown' 时直接插入渲染后的 HTML（现有 `html` 格式相同路径）

### 3.4 批量导入进度

**改动**：
- `POST /ingest/batch` 端点：接受 `paths`（文件路径列表），调用 `ingest_ebook` 循环，通过 SSE 逐文件报告进度
- 使用现有 SSE 基础设施（`_send_sse` / `do_POST` 的 `stream: true` 分支）
- SSE 事件：`event: progress` `data: {"current": 1, "total": 10, "file": "xxx.pdf", "status": "ok|error"}`
- CLI 保持原样，不涉及批量进度条

### 3.5 测试

- 标签：CRUD 测试、唯一约束、空数据、搜索过滤
- 收藏：toggle 测试、列表、重复 toggle
- Markdown：`markdown` 库可用/不可用两条路径
- 批量导入：mock ingest_ebook 验证 SSE 进度事件

## 4. 横切约定

- 新增 `doc_tags` / `favorites` 表接 `install_triggers`（WAL 复制）
- PII 不涉及（标签/收藏不包含患者数据）
- 所有 LLM 调用走 `get_provider()`，`NoOpProvider` 退回占位
