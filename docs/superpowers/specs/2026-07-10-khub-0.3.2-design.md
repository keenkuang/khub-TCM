# 0.3.2 设计说明 — 数据隔离

## 方案

在 `khub/auth.py` 中新增 `scope_filter(user, resource)` 函数，返回 SQL WHERE 子句片段。在 api.py 的各端点查询中应用该片段。

## 隔离规则

| 角色 | patients | records | consultations | appointments |
|------|----------|---------|---------------|--------------|
| admin | 全部 | 全部 | 全部 | 全部 |
| doctor | 自己接诊的 | 自己写的 | 自己问诊的 | 自己排班的 |
| nurse | 自己护理的 | 只读+自己护理 | 只读 | 全部 |
| intern | 上级医生的 | 实习生的 | 只读 | 只读 |
| receptionist | 全部(仅基本信息) | — | — | 全部 |
| patient | 仅自己 | 仅自己 | 仅自己 | 仅自己 |
| guardian | 被监护人 | 被监护人 | 被监护人 | 代预约 |
| security | 全部(仅基本+报警) | — | — | — |

## 实施

`khub/auth.py` 追加 `scope_filter` + 在 `khub/clinical/*.py` / `khub/ops/store.py` 对应函数加 user 参数。
