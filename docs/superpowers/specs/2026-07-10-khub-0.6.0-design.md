# khub 0.6.0 设计说明 — 开放平台与插件系统

## 三大能力

### 插件系统
- `khub/plugins/base.py` — PluginBase 抽象类（`on_startup`/`on_request`/`on_shutdown`）
- `khub/plugins/registry.py` — 插件注册表（发现/加载/生命周期管理）
- `khub/plugins/examples/hello.py` — 示例插件
- 目录 `khub/plugins/` 下任意 `.py` 文件自动发现

### OpenAPI 文档
- `GET /api/openapi.json` — 自动生成 OpenAPI 3.0 规范
- `GET /api/docs` — Swagger UI 页面

### Webhook 事件推送
- `webhook_subscriptions` 表（event, url, secret）
- `webhook_deliveries` 表（事件发送日志）
- `trigger_webhook(event, payload)` 函数
- 在关键业务操作后调用（document created/appointment booked 等）

## 实现

### 1. PluginBase

```python
class PluginBase:
    name: str = ""
    version: str = "0.1.0"
    def on_startup(self, store): pass
    def on_request(self, method, path, body, current_user): return None
    def on_shutdown(self, store): pass
```

### 2. Registry

插件发现：`khub/plugins/*.py` 中扫描 `PluginBase` 子类，实例化并注册。
生命周期：`load_plugins(store)` / `shutdown_plugins(store)`

### 3. OpenAPI

遍历 `dispatch()` 中的 if-elif 链，提取 method/path 和参数，生成 OpenAPI 3.0 JSON。

### 4. Webhook

事件类型：`document.created`、`appointment.created`、`consultation.created`、`followup.due`
投递：POST JSON + `X-Webhook-Signature`（HMAC-SHA256）→ 异步线程
重试：3 次，指数退避（10s/60s/300s）
