## 实施

### 通知表
```sql
CREATE TABLE notifications (
    id INTEGER PRIMARY KEY,
    user_id INTEGER,               -- 目标用户(NULL=全部)
    title TEXT NOT NULL,
    body TEXT,
    event_type TEXT,               -- appointment.created / document.updated ...
    resource_type TEXT,             -- appointment / document / consultation
    resource_id TEXT,               -- 关联资源 ID
    read INTEGER DEFAULT 0,
    created_at TEXT DEFAULT (datetime('now'))
);
```

### SSE 通知端点
`GET /events` — 保持长连接，推送 `text/event-stream`。有新通知时 `event: notification` + JSON data。

### 事件集成
`webhook.trigger()` 同时调用 `notifications.create()` 写入 `notifications` 表。
SSE 连接主动轮询新通知（5 秒间隔心跳+新通知推送）。

### 前端
- 导航栏通知铃铛 `🔔` + 未读数红点
- 点击展开通知列表（最近 10 条）
- 点击通知跳转到关联资源
- `EventSource('/events')` 实时接收并更新未读数
