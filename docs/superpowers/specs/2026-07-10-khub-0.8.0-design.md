# khub 0.8.0 实现计划 — 性能与规模

## 优化清单

### 1. 数据库索引
为所有业务表常用的查询列添加索引（`CREATE INDEX IF NOT EXISTS`），在 `_init_schema` 末尾统一执行。

### 2. 查询缓存
简单内存缓存（`TTLCache`），缓存 `/stats`、`/api/info`、知识图谱查询等高频只读查询结果（默认 5 秒 TTL）。

### 3. 慢查询日志
在 `Store` 中添加 `_profile_sql` 装饰器，记录执行超过 500ms 的 SQL。

### 4. 分页优化
`/search` 端点改为游标分页（keyset pagination）替代 `OFFSET`，避免大页码性能衰减。

## 文件清单

| 文件 | 操作 | 职责 |
|------|------|------|
| `khub/db.py` | 修改 | 加索引 + 慢查询日志装饰器 |
| `khub/cache.py` | 创建 | 简单 TTL 缓存 |
| `khub/api.py` | 修改 | stats/search 端点接入缓存 + 游标分页 |
| `tests/test_perf.py` | 创建 | 性能测试（索引/缓存/分页） |
