# khub 2.0 架构重写规划

## 迁移目标

```
当前 (1.x)                  →  目标 (2.0)
──────────────────────────────────────────────
stdlib http.server          →  FastAPI / Uvicorn
单文件 dispatch() if-elif    →  模块化 Router (25+ 路由文件)
同步 sqlite3                 →  aiosqlite 异步 + 同步兼容层
手动 SSE Handler             →  WebSocket + SSE 原生
manual openapi.py            →  FastAPI auto OpenAPI
ThreadingHTTPServer          →  ASGI (Uvicorn)
单进程部署                   →  Gunicorn + Uvicorn workers
```

## 执行计划

### 阶段 1：新 API 框架（并行运行）
- 创建 `khub/api2/` 目录
- `khub/api2/__init__.py` — FastAPI app 工厂
- `khub/api2/deps.py` — 依赖注入（get_store, get_current_user）
- `khub/api2/routers/health.py` — `/health`, `/api/info`, `/api/i18n`
- `khub/api2/routers/auth.py` — `/auth/login`, `/auth/logout`, `/auth/me`
- 兼容层：`khub/api2/legacy.py` — 代理到旧 `api.py`
- 启动时两个 server 共存：旧 8765 端口 + 新 8766 端口

### 阶段 2：渐进迁移
- 逐个领域迁移路由：文档 → 搜索 → 临床 → 运营 → 课程 → 知识图谱 → AI
- 每个路由文件独立，25+ 文件对应 25 个领域

### 阶段 3：下线旧 API
- 全部路由迁移完成 → 8765 切到 FastAPI → 删除 `khub/api.py`
