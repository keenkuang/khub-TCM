# khub 0.4.1 设计说明 — 小程序客户端

## 架构

小程序作为一个独立前端，通过 HTTP REST API 与 khub 后端通信。

```
[微信小程序] ── HTTPS ──→ [khub REST API /web/* 端点]
                              ↓
                          [khub 后端]
```

## MVP 范围（患者端为主，医生端为辅）

### 患者端
- **登录**：调用 `/auth/login`，token 存 wx storage
- **我的预约**：`GET /ops/appointments?patient_id=用户ID` — 显示预约列表
- **孪生摘要**：`GET /twin/<pid>` — 显示健康摘要 + 时间线 + 辨证脉络
- **健康趋势**：`GET /clinical/trends/<pid>` — 显示体质画像、治疗序列
- **我的问诊**：`GET /clinical/consultations?patient_id=用户ID` — 问诊记录列表

### 医生端
- **患者列表**：`GET /clinical/patients` — 显示患者列表（已 scope 隔离）
- **患者详情**：`GET /twin/<pid>` — 完整孪生摘要
- **预约列表**：`GET /ops/appointments?doctor=用户名` — 今日预约

### 新增后端支持

为确保小程序流畅使用，后端需补充：

| 端点 | 方法 | 说明 |
|------|------|------|
| `GET /api/endpoints` | GET | API 版本/状态端点，供小程序检测连接 |
| `GET /ops/appointments` | GET | 扩展 `?patient_id=` 参数支持 |
| `GET /clinical/consultations` | GET | 新增列表端点（?patient_id=） |

## 文件结构

在小程序端（新目录 `miniapp/`）：
- `app.js` / `app.json` / `app.wxss` — 全局配置
- `pages/login/` — 登录页
- `pages/index/` — 首页（患者/医生切换）
- `pages/appointments/` — 预约列表
- `pages/twin/` — 孪生摘要
- `pages/trends/` — 健康趋势
- `pages/patients/` — 医生端患者列表
- `utils/api.js` — REST API 封装（含 token 注入）
