# khub 0.7.1 实现计划 — 工作流引擎

## 数据模型

### `workflow_definitions`
```sql
CREATE TABLE workflow_definitions (
    id INTEGER PRIMARY KEY, name TEXT NOT NULL, description TEXT,
    steps TEXT NOT NULL,              -- JSON 数组：[{name, type, config, next}]
    version INTEGER DEFAULT 1, active INTEGER DEFAULT 1,
    created_at TEXT DEFAULT (datetime('now')));
```

### `workflow_instances`
```sql
CREATE TABLE workflow_instances (
    id INTEGER PRIMARY KEY, definition_id INTEGER NOT NULL,
    entity_type TEXT, entity_id TEXT, -- 关联业务实体
    status TEXT DEFAULT 'running',    -- running/completed/failed/suspended
    current_step TEXT, context TEXT,  -- context JSON
    history TEXT,                     -- JSON 数组
    created_at TEXT, completed_at TEXT);
```

## 核心模块

| 模块 | 文件 | 职责 |
|------|------|------|
| 定义 CRUD | `khub/workflow/store.py` | workflow 定义/实例的增删查改 |
| 执行引擎 | `khub/workflow/engine.py` | 状态机：步进/完成/失败/分支/并行 |
| 集成 | `khub/workflow/triggers.py` | webhook → 创建工作流实例 |

## 实施

### 引擎核心

```python
def step(instance, definitions_step) -> str:
    """执行一步，返回下一个 step name 或 '__end__'。"""
    step_type = definitions_step["type"]  # auto / condition / parallel / notify
    config = definitions_step.get("config", {})
    if step_type == "auto":
        _run_action(config.get("action"), config.get("params", {}))
    elif step_type == "condition":
        return config["branches"].get(_evaluate(config["expression"]), config["branches"].get("_default"))
    elif step_type == "notify":
        _send_notification(config)
    return definitions_step.get("next", "__end__")
```

### 触发器

webhook 事件 → 匹配 `workflow_definitions` → 创建 `workflow_instances` → 启动。
