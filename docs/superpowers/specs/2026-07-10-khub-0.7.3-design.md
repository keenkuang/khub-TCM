# khub 0.7.3 实现计划 — 离线优先与多端同步

## 三大能力

### PWA 离线支持
- `khub/web/sw.js` — Service Worker（缓存 HTML/CSS/JS + /api/info 离线回退）
- `khub/web/manifest.json` — PWA 清单（图标 + 名称 + 显示模式）
- `index.html` 注册 SW

### 同步引擎
- `sync_changes` 表：离线变更日志（entity_type, entity_id, action, data, client_id, version）
- `khub/sync2.py`（区别于 sync_engine.py）：推送/拉取变更 + 冲突检测 + 简单合并
- `POST /api/sync/push` — 客户端推送上行变更
- `GET /api/sync/pull?since=timestamp` — 客户端拉取下行的变更

### 多端同步
- `devices` 表：注册的设备（client_id, name, last_sync_at, last_version）
- CLI：`khub sync status` / `khub sync push` / `khub sync pull`
- 同步状态 Web UI 面板
