# khub 0.5.1 实现计划 — 私有化部署与 OEM 产品化

## 文件清单

| 文件 | 操作 | 职责 |
|------|------|------|
| `install.sh` | 创建 | 一键安装脚本（检测OS → 安装依赖 → 配置 DB → 启动服务） |
| `Dockerfile` | 修改 | 多阶段构建优化、健康检查完善 |
| `docker-compose.yml` | 完善 | 生产配置（volumes/网络/重启策略/日志/资源限制） |
| `helm/khub/Chart.yaml` | 创建 | Helm chart 元数据 |
| `helm/khub/templates/` | 创建 | deployment/service/configmap/ingress/pvc 模板 |
| `khub/api.py` | 修改 | GET /api/info 返回系统信息（版本/名称/logo/uptime） |
| `khub/web/index.html` | 修改 | 白标支持（动态标题/logo 从 /api/info 读取） |
| `docs/deployment.md` | 修改 | 补充一键安装 + Helm + 白标配置说明 |
| `docs/config.md` | 修改 | 补充 KHUB_BRAND_NAME / KHUB_BRAND_LOGO 环境变量 |

## 实施

### 1. 一键安装脚本 `install.sh`
- 检测 OS（Ubuntu/Debian/CentOS）
- 安装 Python 3.11+ / pip / SQLite（如缺失）
- 创建 khub 用户 + 数据目录 `~/.khub/`
- `pip install khub[all]`
- 初始化数据库
- 配置 systemd 服务
- 输出管理员密码

### 2. Docker 生产完善
- Dockerfile 多阶段构建（builder → runtime），减小镜像体积
- docker-compose.yml 追加生产配置：restart: always、日志轮转、健康检查间隔、资源限制

### 3. Helm Chart
- 标准结构：Chart.yaml / values.yaml / templates/
- templates：deployment.yaml、service.yaml、configmap.yaml、ingress.yaml、pvc.yaml

### 4. 白标支持
- 环境变量 `KHUB_BRAND_NAME`、`KHUB_BRAND_LOGO`（Base64 图片或 URL）
- `GET /api/info` 返回 `{name, version, logo_url, uptime}`
- index.html 动态读取 `/api/info` 设置页面标题和 logo
